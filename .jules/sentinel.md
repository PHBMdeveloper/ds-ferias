## 2024-05-09 - [CSV Injection Prevention]
**Vulnerability:** Found multiple CSV export endpoints (`app/api/reports/balance/route.ts`, `app/api/reports/adherence/route.ts`, `app/api/reports/acquisition-periods/route.ts`, `app/api/vacation-requests/export/route.ts`) that did not sanitize user-controlled inputs (names, emails, etc.) before writing them to a CSV format. This exposes a Formula Injection (CSV Injection) vulnerability if a user's input starts with `=, +, -, @` and is later opened in Excel.
**Learning:** In a codebase heavily relying on CSV generation via raw string concatenation, it's easy to miss output encoding. The standard practice of escaping quotes/commas isn't enough to prevent formula execution in spreadsheet software.
**Prevention:** Always use a helper function (like the newly added `escapeCsvFormulas` in `lib/csv.ts`) to prepend a single quote (`'`) to any string that begins with a known formula prefix. Ensure this utility is applied to ALL user-controllable fields before concatenation.
