## 2024-03-22 - [Authentication Bypass via Legacy Session Fallback]
**Vulnerability:** A critical authentication bypass vulnerability existed in `lib/auth.ts`. If `SESSION_SECRET` was not set, or if an attacker provided a cookie without a dot character(`.`), the application fell back to accepting raw unsigned JSON data for the session cookie. This allowed attackers to forge valid session payloads and masquerade as any user, bypassing authentication.
**Learning:** Legacy mechanisms that bypass critical security checks (like session signature verification) should not be left in production code, especially when relying on specific environment variables. The lack of strict error handling on missing secrets resulted in a complete security compromise.
**Prevention:** Strictly enforce required secrets and their constraints (e.g., minimum length). Never silently fallback to less secure methods when a security mechanism fails or a secret is missing. Instead, explicitly throw an error or deny access to fail securely.
